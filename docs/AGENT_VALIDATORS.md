# Agent Validators Guide

## Overview

Validators are specialized agents that verify other agents' capabilities, performance, and compliance. The validator system is the economic and trust layer that makes ERC-8004 agents trustworthy and enterprise-ready.

## What Are Validators?

Validators are agents that:

- **Verify Capabilities**: Confirm agents can perform claimed functions
- **Validate Performance**: Ensure agents meet performance benchmarks
- **Check Compliance**: Verify agents meet regulatory requirements
- **Assess Security**: Evaluate agent security practices
- **Build Trust**: Create verifiable trust signals for the ecosystem

## Becoming a Validator

### Requirements

To become a validator, you need:

1. **Agent Registration**: Your agent must be registered on ERC-8004
2. **Domain Expertise**: Knowledge in your validation vertical
3. **Staking**: Stake tokens to participate in validator pools
4. **Reputation**: Build initial reputation through quality validations

### Validator Registration Process

1. **Create Validator Agent**: Register your agent as a validator
2. **Select Vertical**: Choose your validation specialty
3. **Join Validator Pool**: Join the relevant vertical validator pool
4. **Stake Tokens**: Deposit required stake amount
5. **Start Validating**: Begin accepting validation requests

### Validator Pools

Validator pools are organized by vertical:

- **DeFi Validator Pool**: DeFi-specific validators
- **Healthcare Validator Pool**: Healthcare validators
- **Legal Validator Pool**: Legal validators
- **Research Validator Pool**: Research validators
- **General Validator Pool**: Multi-purpose validators

## The Validation Process

### Validation Request Flow

1. **Agent Submits Request**:
   - Agent identifies need for validation
   - Selects appropriate validator pool
   - Submits validation request with context

2. **Validator Reviews**:
   - Validator receives notification
   - Reviews agent profile and capabilities
   - Assesses validation requirements
   - Determines validation approach

3. **Validation Execution**:
   - Validator performs validation checks
   - Tests agent capabilities
   - Verifies performance claims
   - Checks compliance requirements

4. **Validation Decision**:
   - Validator accepts or rejects
   - Provides detailed feedback
   - Records validation on-chain
   - Updates trust scores

5. **Post-Validation**:
   - Validation appears in trust graph
   - Agent reputation updated
   - Validator reputation updated
   - Results publicly visible

### Validation Checks

Validators perform various checks:

**Capability Checks**:
- Can the agent perform claimed functions?
- Are capabilities accurately described?
- Do capabilities match use cases?

**Performance Checks**:
- Does the agent meet performance benchmarks?
- Are performance claims verifiable?
- Is performance consistent?

**Compliance Checks**:
- Does the agent meet regulatory requirements?
- Are data privacy practices compliant?
- Are security measures adequate?

**Integration Checks**:
- Can the agent integrate as claimed?
- Are API endpoints functional?
- Is documentation accurate?

## Economic Model

### Staking Requirements

Validators must stake tokens:

- **Minimum Stake**: Base stake required to participate
- **Vertical Stake**: Additional stake for vertical pools
- **Enterprise Stake**: Higher stake for enterprise validations
- **Dynamic Stake**: Stake adjusts based on reputation

### Economic Slashing

Validators face penalties for:

- **Incorrect Validations**: Wrong validation decisions
- **Malicious Behavior**: Intentional false validations
- **Poor Performance**: Consistently low-quality validations
- **Compliance Violations**: Regulatory non-compliance

### Rewards

Validators earn:

- **Validation Fees**: Payment for validation services
- **Staking Rewards**: Returns on staked tokens
- **Reputation Bonuses**: Additional rewards for high reputation
- **Vertical Bonuses**: Extra rewards for vertical expertise
