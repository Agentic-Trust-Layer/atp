# Vertical Ontologies Guide

## Overview

ATP uses rich vertical ontologies to enable precise agent discovery by capability and industry. These taxonomies organize agents into meaningful categories, making it easy to find the right agent for specific needs.

## What Are Vertical Ontologies?

Vertical ontologies are:

- **Industry Taxonomies**: Organized industry categories
- **Capability Hierarchies**: Structured capability trees
- **Use Case Classifications**: Categorized use cases
- **Domain Knowledge**: Industry-specific knowledge
- **Discovery Framework**: Framework for agent discovery

## Ontology Structure

### Hierarchical Organization

Ontologies are organized hierarchically:

**Level 1: Vertical**
- DeFi, Healthcare, Legal, etc.

**Level 2: Category**
- Trading, Research, Contracts, etc.

**Level 3: Sub-Category**
- Spot Trading, Medical Research, Contract Analysis, etc.

**Level 4: Capability**
- Specific agent capabilities

**Level 5: Use Case**
- Specific use cases

### Cross-Vertical Connections

Ontologies support cross-vertical connections:

- **Related Verticals**: Connected verticals
- **Shared Capabilities**: Capabilities across verticals
- **Cross-Industry Use Cases**: Use cases spanning verticals
- **Integration Points**: Points of integration
